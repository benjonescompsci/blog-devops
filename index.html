What is it?

Dev – Software development

Ops – IT Operations

Primary Goals

  • Speed

  • Automation

  • Reliability

Parts Involved

CI/CD

testing, building, and deploying

e.g. Gitlab, GitHub actions, Jenkins

changes tested and shipped quickly/safely

CI focuses on integrating code changes frequently and testing

CD focuses on continuous delivery/deployment, automatically deploying to production

Security (DevSecOps)

Check security with CI/CD

e.g. static analysis, dependency checking, vulnerability scanning, ensuring secure coding practices.

e.g. OWASP ZAP, Snyk, Aqua Security

Hosting

Cloud infrastructure for servers and resources

e.g. AWS, Azure, Google Cloud

AWS explained

  • EC2 (virtual servers)

  • S3 (object storage)

  • IAM (permissions)

  • VPC (virtual networking to isolate resources)

IaC (Infrastructure as Code)

Automate cloud infrastructure using code instead of manually setting up a system and environment

You declare what your e.g. server/servers (maybe load balancing)/database should look like and it is created when needed, you don’t modify the infrastructure itself for changes but instead the code used to generate it

Ensures Consistency, Speed, and Scalability of your infrastructure

Allows Infrastructure state to be version controlled and rolled back if needed

e.g. Terraform Ansible CloudFormation

Scaling

You infrastructure can automatically (autoscaling) add or remove servers (horizontal scaling) or add/remove RAM/Compute (vertical scaling) depending on current demand/traffic

  • Efficient

    • Only use what is needed to save money, everything utilised

  • Performant

    • Your app grows to meet any demand

Horizontal scaling is preferred as it is more cost effective and flexible

Containers & Orchestration

You want to deploy apps in containers for consistency and isolation

But you need to orchestrate between containers if you are using multiple, and you need to minimise downtime when updating containers, there are tools to do this automatically

e.g. you have containers for:

  • Front end

  • Back end

  • Database

  • Redundancy

Some of these need to communicate with others e.g. front and back end, some need to step in if another crashes

e.g. Docker, Kubernetes

Reliability

Must maximise uptime by preventing going down and recovering fast

Redundancy

  • Having your infrastructure load balance over multiple servers

  • Backups

Self-Healing

Automatically restarting containers that have crashed (Detected by monitoring)

Monitoring/Logging

catch bugs or performance issues early

e.g. Prometheus (metrics) + Grafana (visualising), ELK stack for data interpretation (Elasticsearch, Logstash, Kibana), Datadog or New Relic

Planning & collaboration

dev and ops teams collaborate

Tools like JIRA or Trello for project tracking.

Generally agile techniques promote this approach

Git VC and Collaboration

e.g. GitHub, collaboration via

  • pull requests

  • code reviews

  • CI/CD integration
